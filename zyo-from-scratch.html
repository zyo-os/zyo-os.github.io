ZYO From Scratch

Guia completo para criar sua própria distribuição Linux personalizada baseada em Debian.

O que você vai aprender:

  • Criar um sistema Debian mínimo com debootstrap
  • Instalar e configurar o ambiente de desktop GNOME ou XFCE
  • Adicionar extensões personalizadas
  • Personalizar logos, papéis de parede e temas visuais
  • Configurar segurança (DNS seguro e Firewall)
  • Gerar ISO bootável

Requisitos e Informações:

  • Tempo estimado: 30 minutos
  • Espaço em disco: Mínimo 5 GB recomendado
  • Conexão de rede: Internet estável é essencial
  • Sistema base: Debian 12+
  • Privilégios: Acesso root ou sudo necessário

LEIA ANTES DE PROSSEGUIR:

  1. Você deve estar utilizando um sistema Linux com base no Debian ou derivados, como Ubuntu, Zorin OS, Linux Mint ou outros.
  2. Conhecimento mínimo em terminal Linux é indispensável para seguir este guia sem cometer erros.
  3. Dar comandos em terminal pode ser perigoso caso não saiba o que está fazendo, então certifique-se antes do ENTER.
  4. Em caso de dúvidas, não realize os procedimentos descritos no tutorial sem antes pesquisar profundamente o que cada coisa faz.
  6. SOBRE O PROJETO ZYO-OS:
  7. O ZYO-OS é um projeto de distribuição Linux baseado no Debian, utilizando-se de tecnologia open-source (código aberto).
  8. O objetivo principal é fornecer um sistema operacional seguro, voltado às necessidades de usuários como crianças e pessoas idosas e/ou que não possuem muita afinidade com a tecnologia.
  9. Pretendemos tornar a vida digital das pessoas mais segura, com foco em privacidade, segurança, e sem utilizar nenhum tipo de IA acoplada ao sistema.
  10. Este guia permite uma instalação personalizada do ZYO-OS. No entanto, você deve baixar o arquivo .iso diretamente do nosso repositório oficial no Sourceforge se desejar o sistema ZYO-OS completo e pré-configurado.
  11. O objetivo deste guia é ensinar como criar uma distro Linux de forma simples, utilizando uma base sólida como o Debian puro. É uma introdução prática ao processo de criação de sistemas. Para quem busca um entendimento profundo (compilando cada componente), recomendamos o estudo do projeto Linux From Scratch (LFS).

Parte 1: Preparação do Ambiente

Vamos começar criando a estrutura de trabalho e instalando as ferramentas necessárias.

1.1

Criar Diretório de Trabalho

Cria a pasta onde todo o trabalho será realizado. Este diretório conterá o sistema Debian mínimo, arquivos de ISO e configurações temporárias.

Comando do terminal 
cd ~ && mkdir -p zyo-os-build && cd zyo-os-build
1.2

Instalar Ferramentas Necessárias

Função de cada ferramenta:

  • debootstrap – Instala um sistema Debian mínimo (base)
  • squashfs-tools – Compacta o sistema em um arquivo comprimido
  • xorriso – Ferramenta para criar e modificar arquivos ISO
  • isolinux/grub – Bootloaders para BIOS e UEFI
  • mtools/dosfstools – Utilitários para criar partições EFI

Aviso:

Este comando pode levar de 2-5 minutos dependendo de sua conexão

Comando do terminal 
sudo apt update && sudo apt install -y debootstrap squashfs-tools xorriso isolinux syslinux-efi grub-pc-bin grub-efi-amd64-bin mtools dosfstools

Parte 2: Criar Sistema Base

Nesta fase, criaremos um sistema Debian completo usando debootstrap. Este processo inclui criação da estrutura de raiz, montagem de sistemas de arquivos especiais e configuração inicial do novo sistema.

2.1

Executar Debootstrap

O debootstrap baixa e instala um sistema Debian completo (porém mínimo) em um diretório. Este é um processo crítico e não deve ser interrompido.

⚠️ NÃO INTERROMPA ESTE PROCESSO!

  • Tempo estimado: 5-15 minutos
  • Download: Aproximadamente 300MB
  • Conexão: Requer acesso à internet

Parâmetros utilizados:

  • --arch=amd64 – Arquitetura de 64 bits (padrão)
  • --variant=minbase – Instala apenas pacotes essenciais
  • --include=systemd-sysv – Gerenciador de init (necessário para boot)
  • bookworm – Versão estável atual do Debian
Comando do terminal 
sudo debootstrap --arch=amd64 --variant=minbase --include=systemd-sysv bookworm chroot http://deb.debian.org/debian/
2.2

Montar Sistemas de Arquivos Virtuais

Para gerenciar o novo sistema com chroot, precisamos montar diretórios especiais (pontos de montagem virtuais) que permitem acesso a dispositivos, processos e informações do kernel.

Diretórios montados:

  • /dev – Dispositivos (disco, USB, console)
  • /dev/pts – Pseudo-terminais (para shells interativos)
  • /proc – Informações de processos e sistema
  • /sys – Parâmetros do kernel e drivers
Comando do terminal 
sudo mount --bind /dev chroot/dev && sudo mount --bind /dev/pts chroot/dev/pts && sudo mount --bind /proc chroot/proc && sudo mount --bind /sys chroot/sys
2.3

Entrar no Ambiente Chroot

chroot significa "change root" – você estará assumindo a raiz de uma árvore de diretório diferente. A partir deste ponto, você estará configurando o novo sistema como se estivesse em um computador separado.

📌 IMPORTANTE!

Após executar este comando, o prompt vai mudar para: root@seu-pc:/#

Você está DENTRO do novo sistema! Qualquer comando será executado no novo Debian, não em seu sistema anfitrião.

Comando do terminal 
sudo chroot chroot /bin/bash
2.4

Configurar Variáveis de Ambiente

Essas variáveis definem como o sistema se comportará durante a instalação, evitando prompts interativos e mensagens desnecessárias.

Variáveis explicadas:

  • HOME=/root – Define o diretório home do usuário root
  • LC_ALL=C – Define localização para inglês (evita problemas de codificação)
  • DEBIAN_FRONTEND=noninteractive – Aceita valores padrão sem perguntas
Comando do terminal 
export HOME=/root && export LC_ALL=C && export DEBIAN_FRONTEND=noninteractive
2.5

Configurar Repositórios de Pacotes

Configura a origem dos pacotes que serão instalados. Os repositórios definem de onde o Debian baixará software.

Categorias de repositórios:

  • main – Software livre oficial do Debian
  • contrib – Software livre que pode depender de componentes não-livres
  • non-free – Software proprietário ou com restrições de licença
  • non-free-firmware – Drivers de hardware e microcode
Comando do terminal 
cat > /etc/apt/sources.list << 'EOF'
deb http://deb.debian.org/debian/ bookworm main contrib non-free non-free-firmware
deb http://deb.debian.org/debian/ bookworm-updates main contrib non-free non-free-firmware
deb http://security.debian.org/debian-security bookworm-security main contrib non-free non-free-firmware
EOF
2.5.1

Atualizar Lista de Pacotes

Baixa a lista atualizada de pacotes dos repositórios configurados. Necessário fazer após alterar os repositórios.

O que faz:

apt update sincroniza a base de dados local com os repositórios remotos

Comando do terminal 
apt update
2.6

Configurar Hostname e Resolução de Nome

Define o nome da máquina e configura como o sistema resolve nomes de rede local.

Dica:

Você pode trocar zyo-os por qualquer nome que desejar (exemplo: meu-linux, debian-custom)

O que está sendo configurado:

  • /etc/hostname – Nome identificador da máquina
  • /etc/hosts – Mapeamento local de nomes para IPs (sem necessidade de DNS)
Comando do terminal 
echo 'zyo-os' > /etc/hostname && cat > /etc/hosts << 'EOF'
127.0.0.1   localhost
127.0.1.1   zyo-os
::1         localhost ip6-localhost ip6-loopback
EOF

Configurar Localização de Idioma e Teclado

Define o idioma, codificação de caracteres, formato de data/hora e layout de teclado do sistema. Isso afeta menus, mensagens de erro e outputs de comandos.

Seletor de Localização e Teclado

Use o formulário abaixo para selecionar sua localização e gerar o comando personalizado:

Comando Personalizado Gerado:

Locales (localizações) comuns:

Parte 3: Kernel e Utilitários de Boot

O kernel é o núcleo do sistema operacional. Nesta fase, instalaremos o kernel Linux e ferramentas essenciais para que o sistema possa inicializar e rodar a partir de uma ISO/USB.

3.1

Instalar Kernel Linux

O kernel é o programa que gerencia recursos de hardware (CPU, memória, disco) e permite que o sistema operacional funcione. Também instalamos os headers necessários para compilação de módulos.

Informações:

  • Tempo estimado: 3-5 minutos
  • Tamanho: Aproximadamente 100MB
Comando do terminal 
apt install -y linux-image-amd64 linux-headers-amd64
3.2

Instalar Utilitários de Boot em Live (CRÍTICO!)

EXTREMAMENTE IMPORTANTE!

Sem este pacote, o sistema vai travar em uma tela preta!

O live-boot detecta que está rodando de um CD/USB/PenDrive e configura tudo automaticamente. Sem ele, o sistema tenta acessar um disco rígido que não existe.

Pacotes instalados:

  • live-boot – Utilitários de boot ao vivo
  • live-boot-initramfs-tools – Suporte para inicialização RAM
  • live-config – Configuração automática em tempo de boot
  • live-config-systemd – Integração com init system moderno
Comando do terminal 
apt install -y live-boot live-boot-initramfs-tools live-config live-config-systemd
3.3

Instalar Firmwares de Hardware

Firmwares são programas pequenos que controlam hardware específico. Incluem drivers para placas WiFi, Ethernet, impressoras e outros periféricos.

Firmwares instalados:

  • firmware-linux – Drivers de hardware genéricos
  • firmware-realtek – Placas de rede Realtek
  • firmware-atheros – Placas WiFi Atheros
  • firmware-iwlwifi – Placas WiFi Intel
Comando do terminal 
apt install -y firmware-linux firmware-linux-nonfree firmware-misc-nonfree firmware-realtek firmware-atheros firmware-iwlwifi
3.4

Instalar Gerenciador de Rede

NetworkManager facilita conexão com redes WiFi e Ethernet. Com este pacote, o sistema conectará automaticamente a redes conhecidas.

Ferramentas de rede instaladas:

  • network-manager – Gerenciador de conexões principal
  • wireless-tools – Utilitários para WiFi
  • wpasupplicant – Suporte para redes WPA/WPA2
  • net-tools – Ferramentas clássicas de rede (ifconfig, route)
Comando do terminal 
apt install -y network-manager wireless-tools wpasupplicant net-tools && systemctl enable NetworkManager

Parte 4: Configurar Usuários e Permissões

Criar usuários não-root e configurar permissões é essencial para segurança e usabilidade do sistema.

4.1

Criar Usuário para Modo Live

Cria um usuário padrão para que as pessoas usem o sistema. Não é recomendado rodar tudo como root (super-usuário).

Configuração padrão:

  • Usuário: liveuser
  • Senha: live
  • Grupos: sudo (permissões administrativas), netdev (rede), audio, video, plugdev (USB)

Dica: Você pode trocar essas credenciais se desejar

Comando do terminal 
apt install -y sudo && useradd -m -s /bin/bash -G sudo,netdev,audio,video,plugdev liveuser && echo 'liveuser:live' | chpasswd && echo 'root:root' | chpasswd
4.2

Configurar Sudo Sem Senha

Permite que o usuário liveuser execute comandos com privilégios de administrador sem digitar senha. Isso melhora a experiência em modo live.

Segurança:

Isso é adequado para um live USB/CD, mas nunca faça isso em um sistema permanente conectado à internet!

Comando do terminal 
echo 'liveuser ALL=(ALL:ALL) NOPASSWD: ALL' > /etc/sudoers.d/liveuser && chmod 0440 /etc/sudoers.d/liveuser

Comando do terminal

[security]
[xdmcp]
[chooser]
[debug]
EOF

5.7

Instalar Extensões Personalizadas do GitHub

Ampliar a funcionalidade do GNOME instalando extensões personalizadas diretamente do GitHub. Perfeito para adicionar recursos que não estão nos repositórios padrão.

Como Funcionam as Extensões do GitHub

A maioria das extensões do GNOME segue a mesma estrutura:

  • UUID: Identificador único (ex: blur-my-shell@aunetx)
  • Localização: Instaladas em /usr/share/gnome-shell/extensions/
  • Ativação: Registradas na configuração dconf
  • Nomeação: A pasta deve usar exatamente o UUID

Importante:

Nem todas as extensões são compatíveis com sua versão do GNOME. Verifique a compatibilidade no repositório antes de instalar.

Exemplo Prático: Instalar "Blur My Shell"

Uma extensão popular que adiciona efeito de desfoque ao GNOME.

Comando do terminal 
apt install -y git && cd /tmp && git clone https://github.com/aunetx/blur-my-shell && cd blur-my-shell && mkdir -p /usr/share/gnome-shell/extensions/ && cp -r blur-my-shell@aunetx /usr/share/gnome-shell/extensions/

Agora ativar a extensão na configuração:

Comando do terminal 
dconf write /org/gnome/shell/enabled-extensions "['blur-my-shell@aunetx']"

Pronto!

Reinicie o GNOME Shell: killall -HUP gnome-shell

Instalando Múltiplas Extensões

Se quiser adicionar várias extensões de uma vez, combine-as na lista:

Comando do terminal 
dconf write /org/gnome/shell/enabled-extensions "['blur-my-shell@aunetx', 'dash-to-dock@micxgo.gmail.com', 'user-theme@gnome-shell-extensions.gcampax.github.com']"

Onde Encontrar Extensões

  • GNOME Extensions Oficial - Repositório oficial com avaliações
  • GitHub - Busque por "gnome-shell-extension"
  • apt search gnome-shell-extension - Pacotes via repositórios Debian

Dica Extra:

Para listar extensões atualmente ativadas, use:

dconf read /org/gnome/shell/enabled-extensions

Parte 8: Segurança e Proteção de Rede

Configuraremos DNS seguro (para proteção contra malware) e firewall (para bloquear conexões não autorizadas).

8.1

Configurar DNS Seguro (Cloudflare Family)

DNS (Domain Name System) é o "tradutor" da internet. Quando você digita google.com, o DNS converte para um número (IP). Um DNS seguro filtra sites maliciosos, protegendo seu sistema contra malware, trojans e phishing.

Por que usar Cloudflare Family?

Cloudflare Family é um DNS gratuito que oferece proteção em tempo real:

  • Malware: Bloqueia downloads e sites com malware/trojans
  • Phishing: Identifica e bloqueia sites fraudulentos
  • Conteúdo impróprio: Filtra sites adultos (configurável)
  • Rápido e privado: Sem logs de navegação, sem censura

Endereços DNS:

  • 1.1.1.3 – DNS primário (com filtro)
  • 1.0.0.3 – DNS secundário (redundância)
Comando do terminal 
mkdir -p /etc/NetworkManager/conf.d && cat > /etc/NetworkManager/conf.d/dns-servers.conf << 'EOF'
[global-dns-domain-*]
servers=1.1.1.3,1.0.0.3
EOF
8.2

Instalar e Ativar Firewall (UFW)

Um firewall é uma barreira de segurança que controla quais conexões são permitidas. Por padrão, deixamos tudo passar, o que é arriscado. UFW inverte essa lógica: bloqueia tudo e só permite o necessário.

O que acontece sem firewall?

  • Ataques de força bruta: Tentativas de acessar SSH/RDP
  • Serviços abertos: Aplicações pode estar expostas na rede
  • Worms: Malware que se propaga pela rede local

Como funciona o UFW:

  • default deny incoming – Bloqueia TODAS as conexões de entrada (você escolhe quais permitir)
  • default allow outgoing – Permite TODAS as conexões de saída (seu PC pode fazer requisições)
  • --force enable – Ativa imediatamente sem perguntar
  • systemctl enable ufw – Inicia automaticamente na próxima inicialização
Comando do terminal 
apt install -y ufw && ufw default deny incoming && ufw default allow outgoing && ufw --force enable && systemctl enable ufw

Parte 10: Criador de Instalador Gráfico

Crie um instalador interativo com interface gráfica que permite aos usuários instalarem sua distribuição em disco rígido.

Visão Geral do Instalador

O instalador é um script bash interativo que oferece:

  • Seleção de disco – Escolher onde instalar
  • Particionamento automático – EFI (512MB) + raiz
  • Cópia do sistema – Transferir arquivos
  • Instalação do GRUB – Bootloader UEFI
  • Configuração de usuário – Criar conta com senha
  • Interface Zenity – Diálogos gráficos amigáveis
9.1

Criar Diretório e Arquivo do Instalador

Cria a pasta de binários do sistema e prepara o arquivo do instalador.

Localização Padrão:

/usr/local/bin/ é onde você coloca scripts executáveis para o sistema inteiro

Comando do terminal 
mkdir -p /usr/local/bin
9.2

Criar o Script do Instalador

Escreve o script bash completo com interface Zenity no arquivo.

Importante:

Copie TODO o conteúdo sem truncar! O script é longo mas deve estar completo.

Comando do terminal 
cat > /usr/local/bin/zyo-installer.sh << 'EOFINSTALLER'
#!/bin/bash
if [ "$EUID" -ne 0 ]; then exec pkexec "$0"; exit 1; fi
zenity --question --width=500 --title="Instalador ZYO-OS" --text="Bem-vindo ao instalador!\n\nATENÇÃO: A instalação vai APAGAR todos os dados do disco selecionado!\n\nDeseja continuar?" || exit 0
DISKS=$(lsblk -ndo NAME,SIZE,TYPE | grep disk | awk '{print "/dev/" $1 "|" $2}')
DISK_LIST=""
while IFS='|' read -r disk size; do DISK_LIST="$DISK_LIST FALSE $disk $size"; done <<< "$DISKS"
TARGET_DISK=$(zenity --list --radiolist --title="Selecionar Disco" --text="TODOS OS DADOS SERÃO APAGADOS!\n\nSelecione o disco:" --width=500 --height=400 --column="" --column="Disco" --column="Tamanho" $DISK_LIST)
[ -z "$TARGET_DISK" ] && exit 0
zenity --question --width=500 --title="CONFIRMAR INSTALAÇÃO" --text="DISCO: $TARGET_DISK\n\nTODOS OS DADOS SERÃO PERMANENTEMENTE APAGADOS!\n\nTem certeza?" || exit 0
USER_PASS=$(zenity --password --title="Senha do Usuário" --text="Digite a senha para o usuário:")
[ -z "$USER_PASS" ] && USER_PASS="user"
(
echo "10"; echo "# Particionando disco..."
parted -s $TARGET_DISK mklabel gpt
parted -s $TARGET_DISK mkpart primary fat32 1MiB 512MiB
parted -s $TARGET_DISK set 1 esp on
parted -s $TARGET_DISK mkpart primary ext4 512MiB 100%
echo "20"; echo "# Formatando partições..."
mkfs.fat -F32 ${TARGET_DISK}1
mkfs.ext4 -F ${TARGET_DISK}2
echo "30"; echo "# Montando partições..."
mount ${TARGET_DISK}2 /mnt
mkdir -p /mnt/boot/efi
mount ${TARGET_DISK}1 /mnt/boot/efi
echo "40"; echo "# Copiando sistema (isso pode demorar)..."
rsync -aAX / /mnt --exclude={/dev/*,/proc/*,/sys/*,/tmp/*,/run/*,/mnt/*,/media/*,/lost+found,/cdrom}
echo "90"; echo "# Instalando GRUB..."
mount --bind /dev /mnt/dev
mount --bind /proc /mnt/proc
mount --bind /sys /mnt/sys
chroot /mnt grub-install --target=x86_64-efi --efi-directory=/boot/efi --bootloader-id=GRUB
chroot /mnt update-grub
echo "95"; echo "# Configurando usuário..."
chroot /mnt useradd -m -s /bin/bash -G sudo user 2>/dev/null || true
echo "user:$USER_PASS" | chroot /mnt chpasswd
umount /mnt/dev /mnt/proc /mnt/sys /mnt/boot/efi /mnt
echo "100"; echo "# Instalação concluída!"
) | zenity --progress --title="Instalando ZYO-OS" --percentage=0 --auto-close --width=500
zenity --info --width=400 --text="Instalação Concluída!\n\nUsuário: user\nSenha: (a que você definiu)\n\nVocê pode reiniciar agora."
if zenity --question --text="Reiniciar agora?"; then reboot; fi
EOFINSTALLER

Parte 11: Gerar ISO Final e Preparar Distribuição

Nesta fase final, criaremos o arquivo ISO bootável que pode ser gravado em DVD ou USB e será distribuído para os usuários.

10.1

Limpar Sistema de Arquivos Temporários

Remove arquivos temporários, cache de pacotes e histórico de comandos para reduzir significativamente o tamanho da ISO final.

O que será removido:

  • Pacotes de instalação desnecessários
  • Cache APT (gestor de pacotes)
  • Arquivos temporários em /tmp e /var/tmp
  • Histórico de bash do root
Comando do terminal 
apt autoremove -y && apt clean && rm -rf /tmp/* /var/tmp/* /var/cache/apt/archives/*.deb /var/lib/apt/lists/* /root/.bash_history
10.2

Sair do Ambiente Chroot

Volta do ambiente isolado para o seu sistema anfitrião.

Confirmação:

O prompt voltará ao normal. Você não estará mais dentro do chroot.

Comando do terminal 
exit
10.3

Desmontar Sistemas de Arquivos Virtuais

Desmonta os filesystems especiais que foram montados no início (dev, proc, sys).

Comando do terminal 
sudo umount -lf chroot/dev/pts && sudo umount -lf chroot/dev && sudo umount -lf chroot/proc && sudo umount -lf chroot/sys
10.4

Criar Estrutura de Diretórios da ISO

Estrutura que será incluída na ISO com bootloader, kernel e sistema de arquivos comprimido.

Diretórios criados:

  • live/ – Contém kernel e sistema comprimido
  • isolinux/ – Bootloader BIOS
  • boot/grub/ – Bootloader UEFI
Comando do terminal 
mkdir -p image/{live,isolinux,boot/grub}
10.5

Copiar Kernel e Imagem de Inicialização

Copia o kernel Linux e a imagem de inicialização (initrd) do chroot para a estrutura da ISO.

O que é copiado:

  • vmlinuz – Kernel Linux comprimido
  • initrd.img – Imagem de ramdisk (carrega drivers básicos antes do sistema)
Comando do terminal 
sudo cp chroot/boot/vmlinuz-* image/live/vmlinuz && sudo cp chroot/boot/initrd.img-* image/live/initrd.img
10.6

Comprimir Sistema com SquashFS

Comprime todo o sistema de arquivos em um arquivo SquashFS. Este é o arquivo principal que contém sua distribuição completa.

Parâmetros:

  • -comp xz – Melhor compressão possível
  • -b 1M – Tamanho de bloco (1MB)
  • -noappend – Cria novo arquivo

PROCESSO MAIS DEMORADO DA CONSTRUÇÃO!

  • Tempo estimado: 10-20 minutos
  • CPU: Será totalmente utilizada
  • Sistema: Pode ficar lento durante este processo
Comando do terminal 
sudo mksquashfs chroot image/live/filesystem.squashfs -comp xz -b 1M -noappend
10.7

Criar Menu ISOLINUX (Bootloader BIOS)

Configura o menu de boot que aparece quando o usuário inicia a ISO em computadores com BIOS legado.

Dica:

Troque ZYO-OS pelo nome da sua distribuição em todos os labels

Comando do terminal 
cat > image/isolinux/isolinux.cfg << 'EOF'
UI menu.c32
PROMPT 0
TIMEOUT 100
MENU TITLE ZYO-OS Boot Menu

LABEL live
    MENU LABEL ^ZYO-OS Live
    KERNEL /live/vmlinuz
    APPEND initrd=/live/initrd.img boot=live quiet splash

LABEL live-safe
    MENU LABEL ZYO-OS Live (^Safe Mode)
    KERNEL /live/vmlinuz
    APPEND initrd=/live/initrd.img boot=live nomodeset quiet splash

LABEL live-text
    MENU LABEL ZYO-OS Live (^Text Mode)
    KERNEL /live/vmlinuz
    APPEND initrd=/live/initrd.img boot=live text
EOF
10.8

Copiar Arquivos do Bootloader ISOLINUX

Copia os componentes necessários do ISOLINUX para que computadores com BIOS legado possam inicializar.

Arquivos copiados:

  • isolinux.bin – Bootloader principal
  • ldlinux.c32 – Sistema de carregamento
  • menu.c32 – Sistema de menu
  • libutil.c32, libcom32.c32 – Bibliotecas do sistema
Comando do terminal 
sudo cp /usr/lib/ISOLINUX/isolinux.bin image/isolinux/ && sudo cp /usr/lib/syslinux/modules/bios/ldlinux.c32 image/isolinux/ && sudo cp /usr/lib/syslinux/modules/bios/menu.c32 image/isolinux/ && sudo cp /usr/lib/syslinux/modules/bios/libutil.c32 image/isolinux/ && sudo cp /usr/lib/syslinux/modules/bios/libcom32.c32 image/isolinux/
10.9

Criar Menu GRUB (Bootloader UEFI)

Configura o menu de boot para computadores modernos com UEFI/EFI.

Sobre GRUB:

GRUB é um bootloader mais moderno que oferece menu navegável e suporte a múltiplos sistemas operacionais

Dica:

Customize o texto "ZYO-OS Live" para o nome da sua distribuição

Comando do terminal 
cat > image/boot/grub/grub.cfg << 'EOF'
set default="0"
set timeout=10

menuentry "ZYO-OS Live" {
    linux /live/vmlinuz boot=live quiet splash
    initrd /live/initrd.img
}

menuentry "ZYO-OS Live (Safe Mode)" {
    linux /live/vmlinuz boot=live nomodeset quiet splash
    initrd /live/initrd.img
}

menuentry "ZYO-OS Live (Text Mode)" {
    linux /live/vmlinuz boot=live text
    initrd /live/initrd.img
}
EOF

Compila o bootloader GRUB em um formato executável que computadores UEFI conseguem reconhecer.

Comando do terminal 
mkdir -p image/EFI/boot && sudo grub-mkstandalone --format=x86_64-efi --output=image/EFI/boot/bootx64.efi --locales="" --fonts="" "boot/grub/grub.cfg=image/boot/grub/grub.cfg"
10.11

Criar Imagem EFI para USB

Cria uma pequena partição EFI que será embutida na ISO para permitir boot em pendrives.

Comando do terminal 
dd if=/dev/zero of=image/efi.img bs=1M count=10 2>/dev/null && mkfs.vfat image/efi.img && mmd -i image/efi.img efi efi/boot && mcopy -i image/efi.img image/EFI/boot/bootx64.efi ::efi/boot/
10.12

Gerar ISO Final 🎉

Cria o arquivo ISO final bootável compatível com BIOS e UEFI.

Último passo!

Após isso sua distribuição estará pronta para ser distribuída!

Informações:

  • Arquivo de saída: ~/zyo-os.iso
  • Tempo estimado: 2-5 minutos

Dica:

Customize -volid "ZYO-OS" com o nome da sua distribuição

Comando do terminal 
cd ~/zyo-os-build && sudo xorriso -as mkisofs \
    -iso-level 3 \
    -full-iso9660-filenames \
    -volid "ZYO-OS" \
    -eltorito-boot isolinux/isolinux.bin \
    -eltorito-catalog isolinux/boot.cat \
    -no-emul-boot \
    -boot-load-size 4 \
    -boot-info-table \
    -isohybrid-mbr /usr/lib/ISOLINUX/isohdpfx.bin \
    -eltorito-alt-boot \
    -e efi.img \
    -no-emul-boot \
    -isohybrid-gpt-basdat \
    -output ~/zyo-os.iso \
    image

Parabéns! ISO Criada com Sucesso!
